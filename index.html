Getting Started

Metrics is super simple. Plugins write data into a key value store, and other plugins then send that data to dashboards or other reporting tools.

First, create an app using Metrics and add a plugin to run and check a service for updates. These plugins are pre-defined metrics so you don't need to figure out how to measure.

var Metrics = require('metrics'); var charges = require('metrics-stripe-charges'); var helpscout = require('metrics-helpscout'); var metrics = new Metrics() .every('5m', charges('your-stripe-key')) .every('1d', helpscout('your-helpscout-key'));

You can get the value for any of your metrics easily:

metrics.on('stripe charges last month', function (val) { geckoboard('widget-id').number(val)); });

Using Metrics

Metrics emits events when it fetches new data from services and calculates the result. You can customize your reporting by writing your own plugins. The code below listens for updates from Stripe, calculates the revenue, and pushes it to Geckoboard when the event fires:

metrics.on('stripe charges last month', function (val) { geckoboard('widget-id').number(val)); });

Maybe you want to send an email to the team every week with a summary of the support tickets closed:

metrics.on('helpscout tickets closed today', function (val) { mail.send({ to: 'team@company.com', subject: 'Support Summary' text: 'We closed ' + val + ' tickets today!' }); });

Segment uses metrics as an internal API to power our dashboards, auto-updating spreadsheets, and other admin services.

Example: Dashboard

Use Metrics to create a dashboard of data about your business. Simply listen for events and send them to apps like Geckoboard, Ducksboard, Leftronic and Librato.

All that's required is to create a plugin to listen to each of the events and send them to your dashboard using their API.

Plugins allow you to add new integrations to your API. They also come bundled with important metrics that you should be measuring so that you don't need to worry about that MRR calculation ever again.
